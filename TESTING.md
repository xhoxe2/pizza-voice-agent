# Перевірка

Рішення перевірено на трьох рівнях — від ізольованих функцій до живого
діалогу через реальний OpenAI Realtime-модель.

## 1. Модульні тести — `pytest`

`tests/test_tools.py` (7 тестів). Перевіряють mock-логику й те, що агент
реєструє всі чотири інструменти. Стан замовлень ізольовано autouse-фікстурою,
тож тести не впливають один на одного.

```bash
pip install -r requirements-dev.txt
pytest -q
# 7 passed
```

Покриття:

| Тест | Що перевіряє |
| --- | --- |
| `test_get_menu_returns_all_items` | повне меню, усі категорії |
| `test_get_menu_filters_by_category_alias` | фільтр за категорією + україномовний аліас |
| `test_item_details_known_and_unknown` | деталі позиції та коректна помилка |
| `test_create_order_computes_total` | правильний підрахунок суми |
| `test_create_order_rejects_unavailable_item` | відмова за недоступною позицією |
| `test_order_status_known_and_unknown` | статус наявного й відсутнього замовлення |
| `test_agent_exposes_all_four_tools` | агент експонує саме 4 інструменти |

## 2. Інтеграція — підключення сервісів

- **LiveKit worker** реєструється у проєкті (`python agent.py dev` →
  `registered worker … url: wss://…livekit.cloud`) — облікові дані валідні.
- **OpenAI** — ключ автентифікується, модель `gpt-realtime-mini` доступна.

## 3. End-to-end діалог через Realtime-модель

`scripts/eval_dialog.py` проганяє повний сценарій ТЗ **текстом через справжню
Realtime-модель** (`input_modality="text"`) і перевіряє, що в розмові
викликаються правильні інструменти. Це той самий рушій, що й у голосі —
мовлення для нього лише інша модальність.

```bash
python scripts/eval_dialog.py
# OK — усі 4 інструменти спрацювали в живому діалозі.
```

Результат прогону:

| Репліка клієнта | Інструмент | Поведінка агента |
| --- | --- | --- |
| «Які піци у вас є?» | `show_menu(category="pizza")` | перелік, ціни словами, позначив недоступну Гавайську |
| «Розкажи про Пепероні» | `item_details("pz2")` | склад, розмір, ціна |
| «Хочу 2 Маргарити, Іван, телефон, адреса» | — | повторив замовлення й попросив підтвердження |
| «Так, підтверджую» | `place_order(...)` | оформив `ORD-103`, озвучив номер, ~40 хв |
| «Статус ORD-101?» | `order_status("ORD-101")` | «Готується», склад, сума |

Транскрипт (скорочено):

```
USER: Які піци у вас є?
  → show_menu({"category": "pizza"})
  AGENT: У нас є такі піци: Маргарита – сто вісімдесят дев'ять гривень,
         Пепероні – двісті двадцять дев'ять… Гавайська тимчасово недоступна.

USER: Розкажи детальніше про Пепероні
  → item_details({"item_id": "pz2"})
  AGENT: Піца «Пепероні»: діаметр тридцять сантиметрів, томатний соус,
         моцарела та пепероні. Ціна – двісті двадцять дев'ять гривень.

USER: Хочу дві Маргарити. Мене звати Іван, телефон 0991112233, адреса Шевченка 5
  AGENT: Дякую, Іване. Повторю: дві «Маргарити», разом триста вісімдесят
         вісім гривень. Підтверджуєте?

USER: Так, підтверджую
  → place_order({"items":[{"id":"pz1","quantity":2}], "customer_name":"Іван", …})
  AGENT: Замовлення оформлено. Номер – О-Ер-Де сто три, доставка ~40 хвилин.

USER: Перевір статус замовлення ORD-101
  → order_status({"order_id": "ORD-101"})
  AGENT: ORD-101 зараз готується: Пепероні та дві Кока-Коли,
         разом триста двадцять сім гривень.
```
